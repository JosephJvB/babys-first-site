Udacity Lesson 1

  • Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

    • The example in the Udacity video was of building a house. So if web development is house, HTML is the structural engineering and CSS is the interior/exterior decoration. I didn't get a great understanding of the DOM from the Udacity videos but it seems like the DOM is the thing that turns the plans laid by HTML and CSS into their end result, so perhaps the DOM is the site manager who takes the HTML/CSS documents, translates them, and produces the website. Sorry if that's not accurate, I'm still hazy on its strict definition. So HTML is the walls and supports of the house, CSS will tell you how big your TV is, how much insulation is in your walls and what colour those walls are, the DOM is the person who organises the installation of the whole project.

  • What is meant by boxifying design?

    • Boxifying design means taking every element of a webpage design and dividing it into boxes. These boxes often have more boxes inside them. For example a pages' header will be denoted as a box, it might have an image element and a text element, so that two more boxes inside the header box. It is by identifying the boxes on a webpage, boxifying design breaks the page down into its elements.

  • What is the box model?

    • As said above, every element on a page is a box. The box model describes the construction of these element-boxes. So each box has margin, border and padding values that define its size and position relitive to the other elements on the page. Margin is the outermost layer, then the border, while padding is how much space between the content of the element and inside edge of the border.
