Javascript Basics

  • How does JavaScript compare to HTML and CSS?

    • To compare JS to HTML and CSS I'll bring back Udacity's metaphor of building a house. If your website is a house then HTML is the walls and structure and CSS decides what colour the walls are, how thick the carpet is. Java Script might be the appliances, washing machine or light switches. Java Script lets the user interact with features of a website, submit information, or perform a calculation. I'm pretty sure you style an input form in CSS but JS provides the mechanics to make it work.

  • Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'.

    • The flow of JS, like CSS goes from top to bottom but there are ways to change that. Loops are sections of code that make the website perform a fucntion until a defined perameter is met. If the perameter can never be met, then the browser crashes! When you brush your teeth the flow might look like [put paste on brush > wet paste > brush teeth > rinse mouth > rinse brush]. If I were programming this action with JS I might create a loop that checks to see if teeth are clean and only if they are clean would I before move to the "rinse mouth" action.

  • Explain the difference between accessing data from arrays and objects.

    • Arrays and objects are both ways of storing data in JS, both are ways of storing multiple values to a variable. Arrays store data in a linear way, each item in an array will corrospond to a number starting from zero. An array with 3 items will contain items numbered 0, 1 and 2. Objects are good at storing non-linear data using "keys" and "variables". An example task in FreeCodeCamp introduced a variable myCar, there was a key for "wheels" and the value was "4". To access the data in an array you must call the correct list-number of an item. To reference data in an object you must call the key of the value you are after.

  • Explain what functions are and why they are useful.

    • A JS function is a piece of code that will perform a repeatable action. The action could be a simple calculation of variables, it is useful because any time the function is called it will excecute the coded action. This means the function doesn't have to be written more than once which is great in streamlining a code.
